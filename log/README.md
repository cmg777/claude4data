# Progress Logs

Session progress logs that preserve context across Claude Code sessions.

## Purpose

Chat sessions can end unexpectedly. When a new Claude session starts, it has no memory of previous work. These logs bridge that gap by documenting:

- What was done in each session
- Current project state
- Next steps and pending tasks

## Format

Log files use the naming convention: `YYYY-MM-DD_HHMM.md`

Each log should include:

1. **Session Goal** - What the user wanted to accomplish
2. **What Was Done** - Tasks completed during the session
3. **Current State** - Project status at end of session
4. **Next Steps** - Pending tasks or recommendations
5. **Notes** - Any important observations
